# AI Agent 落地咨询 · 流程诊断 SOP

> 一套可复制、可交付的「AI Agent 落地诊断」方法论。
> 帮企业在 1-2 周内回答清楚三个问题：**该不该做？做哪里？怎么做？**

---

## 这是什么

我们给企业做 AI Agent 落地咨询时，**第一步永远不是写方案**，而是**做诊断**。

诊断的目的，不是说服客户买单，而是帮客户搞清楚：

- 自己的业务流程里，**哪些环节真的适合 AI Agent**？
- 哪些是伪需求、哪些是真痛点？
- 投入和回报怎么算？
- 技术风险在哪？

**不做诊断直接做方案 = 给客户挖坑给自己埋雷。**

这套 SOP 就是把诊断流程标准化，让任何一个咨询师拿去都能跑出 80 分的诊断报告。

## 6 步诊断流程总览

```
[Step 1] 启动访谈     →  对齐目标、识别决策链
   ↓
[Step 2] 业务流程梳理 →  画出 AS-IS 流程地图
   ↓
[Step 3] AI 机会点识别 →  打分排序，筛出 Top 3
   ↓
[Step 4] 技术可行性评估 →  数据/模型/集成/合规四维度
   ↓
[Step 5] ROI 测算     →  投入 vs 回报，算清账
   ↓
[Step 6] 诊断报告交付 →  老板能看懂、IT 能落地、财务能算账
```

**典型周期：** 1-2 周（视企业规模和访谈密度）
**典型成本：** 5-15 万人民币（看交付深度）
**典型产出：** 一份《AI Agent 落地诊断报告》+ 3-5 个落地候选场景的优先级排序

## 怎么用这套 SOP

### 自查模式（企业内部 IT/CIO）
1. 通读 `01_诊断方法论.md`（30 分钟）
2. 用 `02-04` 三个模板自己跑一遍（半天）
3. 输出一份内部版诊断初稿，决定是否启动外部咨询

### 协作模式（甲方 + 咨询团队）
1. 咨询团队按 `01_诊断方法论.md` 设计项目计划
2. 双方对齐 `02_启动访谈提纲`，安排 5-8 场访谈
3. 用 `03-04` 模板逐步沉淀产出
4. 按 `05_诊断报告模板` 交付

## 这套 SOP 不解决什么

为了避免误用，先说清楚边界：

- ❌ **不解决"AI 战略"问题** —— 我们关注的是落地，不是顶层战略
- ❌ **不替代技术 PoC** —— 诊断结束后，还需要真实数据跑 PoC 验证
- ❌ **不保证"做了就赚钱"** —— 我们能帮你避免大多数坑，但执行风险仍在企业自身
- ❌ **不是"卖产品"的话术包** —— 诊断是中立的，包括建议"不要做 AI"的可能
